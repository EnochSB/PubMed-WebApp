# PubMed API 기반 논문 데이터 수집·분석 웹 애플리케이션
연구자나 실무자가 특정 키워드로 PubMed에 등록된 논문을 매번 수작업으로 검색하고 정리하는 데는 많은 시간이 소요됩니다. 본 프로젝트는 PubMed API를 통해 논문 메타데이터를 자동으로 수집·저장하고, 이를 연도별·저널별·키워드별로 분석 및 시각화하여 연구 동향을 한눈에 파악할 수 있는 대시보드를 제공하는 것을 목표로 합니다. 또한 수집된 데이터를 기반으로 질의응답이 가능한 챗봇 UI를 함께 구현하여, 사용자가 데이터를 보다 직관적으로 탐색할 수 있도록 지원합니다.

## 요구사항
1. PubMed API 검색 조건 입력: 사이드바에 키워드 입력창, 검색 시작 연도, 끝 연도, 최대 수집 논문 수 (최대100개)를 입력할 수 있는 위젯과 수집 버튼을 구현하시오.
2. 논문 메타데이터 수집
   1. 수집 버튼을 클릭하면 사용자가 입력한 조건으로 PubMed에서 PMID를 수집하고, 해당 ID 논문의 제목, Abstract, 저널, 출판연도, 저자 정보를 수집해 DB에 데이터를 저장하시오.
   2. 입력 예시
      1. 검색 키워드 : COVID-19 vaccine
      2. 시작 연도 : 2022
      3. 끝 연도 : 2025
      4. 최대 수집 논문 수 : 100
   3. DB 테이블 필수 필드: pmid(pk), title, abstract, journal, pub_year, authors
   4. 데이터 저장 시 이미 저장된 데이터라면 저장하지 마시오 (중복 저장 방지)
3. 개요 페이지
   1. 현재 DB에 저장된 전체 논문 수, 좌측 사이드바를 통해 방금 수집한 신규 수집한 논문 수, 중복되어 Skip한 저널 수, DB에 저장된 총 저널 수를 출력하고, 연도별 논문 수와
상위 저널 수를 계산해 개요 탭에 시각화하시오.
4. 논문 검색
   1. 논문 목록 탭에선 사용자로부터 검색어, 검색 연도 범위, 저널을 입력받으시오.
   2. 검색 버튼을 클릭했을 때 필터링이 적용되어 해당하는 논문만 출력되도록 구현하시오.
5. 챗봇 탭
   1. 챗봇 탭에 Memory 기능이 탑재된 간단한 챗봇을 구현하시오.
6. 필터 결과 추출
   1. 필터에 따른 논문 목록 조회(필터 예시: 제목, 시작 연도, 끝 연도, 저널)
   2. 논문 목록 탭에서 csv 다운로드 기능을 구현하시오.
7. 미들웨어 구현
   1. 챗봇 기능에 개인 의료 조언, 진단, 처방 관련 질문에는 답변하지 않고 고정된 안내 문구를 반환하는 미들웨어를 구현하시오.
   2. 테스트 질문 : 음주 후 타이레놀 먹어도 되나요?
   3. 고정 안내 문구 : 이 앱은 PubMed 메타데이터 분석용이며, 개인 의료 조언, 진단, 처방 관련 질문에는 답변할 수 없습니다. 의료 관련 결정은 의료 전문가와 상담해 주세요.
